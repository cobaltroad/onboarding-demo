How it works (technical overview)

  1. Customer admin logs in via your existing OAuth provider (leveraging SSO you already have).
  2. Portal fetches roster data from your API (e.g., GET /api/customers/:id/rosters).
  3. Admin makes changes (invite user, update role, deactivate) → portal posts to your API.
  4. Usage stats refresh in real time or on demand, surfacing seat consumption and activity.
  5. AI analyzes usage patterns weekly, generating natural-language insights delivered via email.

🔧 Implementation: Static frontend (GitHub Pages), calls your existing REST API with OAuth tokens. Analytics powered by simple aggregation queries + GPT-4 for insight generation. No new backend needed—just expose roster CRUD and usage log endpoints.
